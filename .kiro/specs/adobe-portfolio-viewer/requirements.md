# Requirements Document

## Introduction

本システムは、Adobe Illustrator（.ai）およびAdobe Photoshop（.psd）で制作された学生の課題作品を一覧・閲覧できるWebアプリケーションである。教員はローカルPC上の変換ツール（CLIまたはローカルWeb UI）を使用してファイルをPNGに変換し、変換済みPNGのみをアップロードする。学生はサムネイルを閲覧でき、教員は優秀作品にバッジを付与できる。約100名の学生が週1回の課題を合計7回提出する運用を想定する。

デプロイ構成: Next.js on Vercel（フロントエンド + API Routes）+ Supabase（DB + Storage）。ファイル変換はサーバーサイドでは行わず、教員のローカルPCで完結する。変換済みPNGのみをSupabase Storageに保存する（約350MB、無料枠内）。元ファイル（.ai/.psd）はアップロードしない。

システムは閲覧用フロントエンド（学生向け・認証不要）、管理用フロントエンド（教員向け・Supabase Auth認証）、ローカル変換ツール（CLI + Web UI）の3つのコンポーネントで構成する。

## Glossary

- **システム**: Adobe Portfolio Viewerアプリケーション全体
- **閲覧フロントエンド**: Next.js（Vercel）上で動作する、学生が作品を閲覧するための公開Webサイト。認証不要
- **管理フロントエンド**: Next.js（Vercel）上で動作する、教員がアップロード・バッジ付与・マスター管理を行うためのWebサイト。Supabase Auth認証
- **変換ツール**: 教員のローカルPCで動作するNode.jsアプリケーション。CLIモードとローカルWeb UIモードの2つの実行方法を提供する
- **CLI\_モード**: ターミナルから`npx convert-works`コマンドで実行する変換ツールのモード
- **Web*UI*モード**: `npx start-converter`で起動し、ブラウザ（localhost:3000）からGUIで操作する変換ツールのモード
- **バックエンド**: Next.js API Routes（Vercel）上で動作するサーバーサイドAPI
- **データベース**: Supabase（PostgreSQL）。学生情報、課題情報、ファイルメタデータ、バッジ情報を保存する
- **ファイルストレージ**: Supabase Storage。変換済みPNGファイルのみを保存する
- **教員**: ファイルの変換・アップロード、バッジ付与、マスターデータ管理を行うユーザー
- **学生**: 閲覧フロントエンドで作品を閲覧するユーザー
- **学籍番号**: 学生を一意に識別する8桁の番号（例: 12345001）。先頭5桁がコースを示す
- **コース**: 学生が所属するグループ（例: 〇〇Aコース、〇〇Bコース）。学籍番号のプレフィックス（先頭5桁）で識別する。管理フロントエンドでのフィルタリングおよび成績エクスポートに使用する。閲覧フロントエンドでは使用しない
- **課題**: 週1回出題される制作課題（合計7回）
- **バッジ**: 作品の優秀さを示すタグ（例:「丁寧な仕事！」「新しいことにチャレンジ！」）
- **バルクアップロード**: 複数の変換済みPNGファイルを一括でアップロードする機能
- **命名規則**: バルクアップロード用のファイル名パターン。{学籍番号}*{氏名}.png（例: 12345001*黒須哲郎.png）

## Requirements

### 要件 1: 管理フロントエンド認証

**ユーザーストーリー:** 教員として、管理フロントエンドにログインし、権限のある教員のみがアップロードやバッジ付与を行えるようにしたい。

#### 受け入れ基準

1. もし「認証されていないユーザーが管理フロントエンドの任意のページにアクセスした」場合、システムはログイン画面にリダイレクトするものとする
2. 「教員が正しいログイン情報（メールアドレスおよびパスワード）を入力する」とき、システムはSupabase Authを通じて教員を認証し、管理ダッシュボード画面を表示するものとする
3. もし「無効なログイン情報が入力された」場合、システムは認証に失敗した旨のエラーメッセージを表示し、アクセスを拒否するものとする。連続5回ログインに失敗した場合、システムは当該アカウントを30分間ロックするものとする
4. 「教員がログアウトする」とき、システムはセッションを終了し、ログイン画面にリダイレクトするものとする
5. もし「認証されていないユーザーが管理APIにアクセスした」場合、バックエンドはリクエストを拒否し、認証エラーを返すものとする
6. もし「教員のセッションが最終操作から60分以上経過した」場合、システムはセッションを無効化し、次回アクセス時にログイン画面にリダイレクトするものとする

### 要件 2: ファイルアップロード（単体）

**ユーザーストーリー:** 教員として、変換済みPNGファイルを1件ずつアップロードし、学生と課題に紐づけたい。

#### 受け入れ基準

1. 「教員が対象の学生と課題を選択し、PNGファイルを選択してアップロードする」とき、バックエンドはファイルを受信し、選択された学生と課題に紐づけてファイルストレージに保存し、メタデータをデータベースに記録するものとする
2. 「教員がファイルをアップロードする」とき、システムはファイルサイズが2MB以下であることを検証するものとする
3. もし「アップロードされたファイルの拡張子が .png 以外である」場合、システムはアップロードを拒否し、対応ファイル形式（.png のみ）を示すエラーメッセージを返すものとする
4. もし「アップロードされたファイルが2MBを超える」場合、システムはアップロードを拒否し、ファイルサイズ上限（2MB）を示すエラーメッセージを返すものとする
5. 「ファイルのアップロードが成功した」とき、管理フロントエンドはアップロードされたファイル名、紐づけられた学生名、および課題名を含むアップロード完了メッセージを教員に表示するものとする
6. もし「教員が指定した学生または課題がデータベースに存在しない」場合、システムはアップロードを拒否し、該当する学生または課題が未登録である旨のエラーメッセージを返すものとする
7. もし「アップロード処理中にバックエンドでエラーが発生した」場合、システムはファイルの保存処理をロールバックし、アップロード失敗を示すエラーメッセージを教員に返すものとする

### 要件 3: バルクアップロード

**ユーザーストーリー:** 教員として、変換ツールで変換済みの約100名分のPNGファイルを一括でアップロードし、ファイル名から各学生に自動紐づけしたい。

#### 受け入れ基準

1. 「教員が複数のPNGファイルを選択する」とき、システムは最大150ファイルまでの一括アップロードを受け付けるものとする
2. 「教員がバルクアップロードを実行する前に課題を選択する」とき、システムは選択された課題を一括アップロード対象の全ファイルに紐づけるものとする
3. 「教員がバルクアップロードを実行する」とき、システムはファイル名のパターン（{学籍番号}*{氏名}.png、例: 12345001*黒須哲郎.png）に基づいて各ファイルを対応する学生に自動紐づけするものとする
4. 「バルクアップロード処理が進行中」の間、管理フロントエンドは処理済みファイル数と全ファイル数を各ファイルの処理完了ごとに更新して教員に表示するものとする
5. もし「バルクアップロード中に一部ファイルの紐づけが失敗した」場合、システムは失敗したファイルのファイル名と失敗理由（ファイル名が命名規則に不一致、学籍番号が未登録、ファイルサイズが2MB超過のいずれか）を一覧表示し、成功したファイルは正常に処理を続行するものとする
6. 「バルクアップロードが完了した」とき、システムは成功件数と失敗件数のサマリーを教員に表示するものとする
7. 管理フロントエンドは、バルクアップロード画面上にファイル命名規則（{学籍番号}*{氏名}.png、例: 12345001*黒須哲郎.png）を常時表示するものとする
8. もし「選択されたファイル数が150を超える」場合、システムはアップロードを受け付けず、最大ファイル数を示すエラーメッセージを表示するものとする
9. もし「アップロードされたファイルの拡張子が .png 以外である」場合、システムは該当ファイルのアップロードを拒否し、PNGファイルのみ対応である旨のエラーメッセージを返すものとする

### 要件 4: ローカル変換ツール

**ユーザーストーリー:** 教員として、ローカルPCで .ai/.psd ファイルをPNGに変換し、バルクアップロード用の正しいファイル名に整形された状態で出力したい。

#### 受け入れ基準

1. 「教員がCLI\_モードで`npx convert-works ./folder --assignment 1`を実行する」とき、変換ツールは指定フォルダ内の全 .ai および .psd ファイルを検出し、変換処理を開始するものとする
2. 「教員がWeb*UI*モードで`npx start-converter`を実行する」とき、変換ツールはlocalhost:3000でローカルWebサーバーを起動し、ブラウザからGUIで操作可能な画面を表示するものとする
3. 「変換ツールが .ai ファイルを検出した」とき、変換ツールはGhostscriptを使用してファイルをPNG形式に変換し、長辺が1280ピクセル以下となるようリサイズするものとする
4. 「変換ツールが .psd ファイルを検出した」とき、変換ツールはag-psdおよびsharpを使用して全レイヤーを統合したうえでファイルをPNG形式に変換し、長辺が1280ピクセル以下となるようリサイズするものとする
5. 「PNGへの変換が完了した」とき、変換ツールは変換後のPNGファイルを500KB以下に圧縮するものとする
6. 「変換ツールがファイルを処理する」とき、変換ツールは元のファイル名を命名規則（{学籍番号}\_{氏名}.png）に適合するか検証し、不適合の場合は教員に修正候補を提示するものとする
7. 「変換処理が進行中」の間、変換ツールは処理済みファイル数、全ファイル数、および現在処理中のファイル名をリアルタイムで表示するものとする
8. 「全ファイルの変換が完了した」とき、変換ツールは変換結果のサマリー（成功件数、失敗件数、出力先フォルダパス）を表示するものとする
9. 「Web*UI*モードで変換が完了した」とき、変換ツールは変換済みPNGのプレビュー一覧をブラウザ上に表示するものとする
10. もし「変換対象フォルダ内に .ai および .psd 以外のファイルが存在する」場合、変換ツールは該当ファイルをスキップし、スキップしたファイル名を教員に通知するものとする
11. もし「変換処理中に個別ファイルでエラーが発生した」場合、変換ツールは該当ファイルのファイル名とエラー内容をログに記録し、残りのファイルの変換処理を続行するものとする
12. もし「Ghostscriptがローカル環境にインストールされていない」場合、変換ツールは起動時にGhostscriptが未インストールである旨のエラーメッセージとインストール手順を表示するものとする

### 要件 5: 作品一覧表示

**ユーザーストーリー:** 学生として、変換済みの作品サムネイルを一覧で閲覧し、他の学生の作品を参考にしたい。

#### 受け入れ基準

1. 「ユーザーが閲覧フロントエンドにアクセスする」とき、閲覧フロントエンドは変換済みPNGのサムネイルをグリッド形式で、課題番号の降順（最新課題が先頭）で表示するものとする
2. 閲覧フロントエンドは、各サムネイルに学生名、課題名、付与されたバッジを表示するものとする。バッジが未付与の作品にはバッジ表示領域を空とするものとする
3. 「ユーザーがサムネイルをクリックする」とき、閲覧フロントエンドは作品の拡大表示を行うものとする。拡大表示にはビューポート幅の最大90%で元の縦横比を維持した画像を表示し、閉じるボタンまたは画像外領域のクリックで一覧表示に戻れるものとする
4. 閲覧フロントエンドは、100名×7課題（最大700件）の作品データを1回あたり20件ずつページネーションまたは遅延読み込みで表示するものとする
5. もし「表示対象の作品が0件である」場合、閲覧フロントエンドは作品が存在しない旨のメッセージを表示するものとする
6. もし「サムネイル画像の読み込みに失敗した」場合、閲覧フロントエンドは該当箇所にプレースホルダー画像を表示し、学生名・課題名の情報は維持するものとする

### 要件 6: フィルタリング機能

**ユーザーストーリー:** ユーザーとして、学生ごと・課題ごと・バッジの有無で作品をフィルタリングし、目的の作品を素早く見つけたい。

#### 受け入れ基準

1. 「ユーザーが学生フィルターから1名以上の学生を選択する」とき、閲覧フロントエンドは選択された学生の作品のみを表示するものとする
2. 「ユーザーが課題フィルターから1つ以上の課題を選択する」とき、閲覧フロントエンドは選択された課題の作品のみを表示するものとする
3. 「ユーザーがバッジフィルターを選択する」とき、閲覧フロントエンドは「バッジ付き作品のみ表示」または特定のバッジ種別が付与された作品のみを表示するものとする
4. 「ユーザーが複数のフィルター条件を組み合わせる」とき、閲覧フロントエンドはすべての条件をANDで結合し、すべての条件を満たす作品のみを表示するものとする
5. もし「フィルター条件に一致する作品が存在しない」場合、閲覧フロントエンドは該当作品がない旨のメッセージを表示し、現在のフィルター条件を維持するものとする
6. 「ユーザーがフィルターを解除する」とき、閲覧フロントエンドはフィルター条件をリセットし、全作品を再表示するものとする
7. 閲覧フロントエンドは、フィルター適用後に該当する作品件数を表示するものとする

### 要件 7: バッジ付与機能

**ユーザーストーリー:** 教員として、優秀な作品にバッジを付与し、学生の成果を可視化したい。

#### 受け入れ基準

1. 「教員が作品を選択してバッジを付与する」とき、システムは選択されたバッジを作品のメタデータに保存し、管理フロントエンドは付与成功を示すフィードバックを教員に表示するものとする
2. システムは、1つの作品に対して最大3個のバッジを付与可能とし、同一バッジの重複付与を禁止するものとする
3. 管理フロントエンドは、定義済みのバッジ一覧（例:「丁寧な仕事！」「新しいことにチャレンジ！」）から教員が選択できるインターフェースを提供するものとする
4. 「教員がバッジを付与した」とき、管理フロントエンドは2秒以内に対象作品のサムネイルにバッジを反映するものとする
5. 「教員が付与済みバッジを削除する」とき、システムは対象のバッジをメタデータから削除し、管理フロントエンドは2秒以内に表示を更新するものとする
6. もし「作品に既に3個のバッジが付与されている状態で教員が追加のバッジを付与しようとした」場合、システムは付与を拒否し、バッジ上限に達している旨のエラーメッセージを表示するものとする

### 要件 8: 学生・課題・コース マスターデータ管理

**ユーザーストーリー:** 教員として、学生名簿・課題情報・コース情報を管理し、アップロード時の紐づけと成績エクスポート時のコース別フィルタリングを正確に行いたい。

#### 受け入れ基準

1. 「教員が学生情報（学籍番号、氏名、ヨミガナ、英字氏名）を登録する」とき、システムは学籍番号が8桁の数字であること、および氏名が1文字以上50文字以下であることを検証し、データベースに学生レコードを作成するものとする。学生レコード作成時、システムは学籍番号の先頭5桁に一致するコースを自動的に紐づけるものとする
2. 「教員が課題情報（課題番号、課題名）を登録する」とき、システムは課題番号が1〜7の整数であること、および課題名が1文字以上100文字以下であることを検証し、データベースに課題レコードを作成するものとする
3. 「教員がコース情報（コース名、学籍番号プレフィックス）を登録する」とき、システムはプレフィックスが5桁の数字であることを検証し、データベースにコースレコードを作成するものとする
4. 「教員が学生情報のCSVファイルをインポートする」とき、システムはCSVファイル（必須カラム: 学籍番号, 学生氏名。任意カラム: ヨミガナ, 英字氏名）を読み込み、最大200行までの学生レコードを一括登録するものとする。CSVファイルはBOM付きUTF-8に対応し、余分なカラム（空カラム含む）は自動的に無視するものとする。各学生の学籍番号先頭5桁から該当コースを自動判定して紐づけるものとする
5. もし「登録しようとする学籍番号が既に存在する」場合、システムは重複エラーを示すメッセージを返し、該当レコードの登録を拒否するものとする
6. もし「登録しようとする課題番号が既に存在する」場合、システムは重複エラーを示すメッセージを返し、該当レコードの登録を拒否するものとする
7. もし「CSVインポート中にバリデーションエラーが発生した行がある」場合、システムはエラーのある行をスキップして残りの行の登録を続行し、完了後にエラー行の一覧とエラー理由を教員に表示するものとする
8. もし「CSVファイルの形式が不正である（必須カラム「学籍番号」「学生氏名」の不足、または200行超過）」場合、システムはインポートを拒否し、原因を示すエラーメッセージを返すものとする
9. もし「学生の学籍番号プレフィックスに一致するコースが登録されていない」場合、システムはコース未設定の状態で学生レコードを作成し、教員にコース未紐づけの旨を通知するものとする

### 要件 9: データ整合性と保存

**ユーザーストーリー:** 教員として、アップロードした全データが安全に保存され、必要な時に参照できること。

#### 受け入れ基準

1. バックエンドは、アップロードされた変換済みPNGファイルをSupabase Storageに保存するものとする。元ファイル（.ai, .psd）はアップロード対象外とする
2. データベースは、各ファイルのメタデータ（学生、課題、アップロード日時、ファイルサイズ、バッジ情報）を記録するものとする
3. もし「データベースへの書き込み中にエラーが発生した」場合、システムはトランザクションをロールバックし、データの不整合を防止するものとする
4. システムは、同一学生・同一課題に対する重複アップロードを検知し、教員に上書きまたはスキップの選択肢を提示するものとする
5. ファイルストレージは、変換済みPNGの合計容量が500MB以下で運用されることを前提とし、Supabase無料枠の範囲内で動作するものとする
